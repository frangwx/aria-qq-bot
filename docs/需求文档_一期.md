# 绝区零 QQ 机器人需求文档（一期）

## 一、项目概述

### 1.1 项目背景
本项目旨在开发一款面向《绝区零》游戏玩家的 QQ 机器人，提供游戏资讯推送、智能对话、游戏信息查询等功能，提升玩家的游戏体验。

### 1.2 项目目标
- 构建一个稳定、可扩展的 QQ 机器人服务
- 提供绝区零游戏相关的资讯推送与查询功能
- 接入大模型实现智能对话，扮演游戏角色"爱芮"
- 支持多用户会话管理与长期记忆

### 1.3 参考项目
- **Miao-Yunzai**: 喵版 Yunzai，源自 Le-niao/Yunzai-Bot V3，提供机器人基础架构参考
- **ZZZ-Plugin**: 绝区零游戏信息获取插件，提供游戏数据接口参考

## 三、功能需求

### 3.1 设置相关

#### 3.1.1 QQ 账户接入

**需求描述**：支持两种 QQ 机器人接入方式

**方案一：腾讯官方 QQ 机器人 API**
- 通过 QQ 开放平台注册机器人
- 获取 AppID 和 AppSecret
- 支持 WebSocket/Webhook 事件订阅
- 支持单聊、群聊、频道场景
- 消息频率限制：
  - 单聊主动消息：每月 4 条/用户
  - 群聊主动消息：每月 4 条/群
  - 被动回复有效期：单聊 60 分钟，群聊 5 分钟

**方案二：OneBot 协议（go-cqhttp）**
- 基于 OneBot v11 协议标准
- 支持 HTTP API、正向/反向 WebSocket
- 社区生态丰富，插件兼容性好
- 需自行处理账号安全风险

**功能要点**：
| 功能 | 优先级 | 说明 |
|------|--------|------|
| 官方 API 接入 | P0 | 推荐方案，合规稳定 |
| OneBot 协议接入 | P1 | 备选方案，功能更灵活 |
| 多账号支持 | P2 | 支持同时接入多个机器人账号 |

#### 3.1.2 主人 QQ 设置

**需求描述**：支持设置机器人主人账号，拥有最高权限

**功能要点**：
| 功能 | 优先级 | 说明 |
|------|--------|------|
| 设置主人 QQ | P0 | 通过配置文件或命令设置 |
| 多主人支持 | P1 | 支持设置多个管理员账号 |
| 权限分级 | P1 | 区分超级管理员、普通管理员、普通用户 |

#### 3.4.7 敏感词过滤

**需求描述**：对话内容需进行敏感词过滤，确保合规

**过滤机制**：
| 层级 | 方法 | 说明 |
|------|------|------|
| 第一层 | 关键词匹配 | 基于敏感词库的直接匹配 |
| 第二层 | 正则匹配 | 识别变体词、谐音词 |
| 第三层 | 语义分析 | 通过 NLP 模型识别隐晦违规 |
| 第四层 | 大模型安全 | 利用模型内置安全机制 |

**敏感词类别**：
- 政治敏感词
- 色情低俗词
- 暴力恐怖词
- 违法犯罪词
- 广告营销词
- 个人隐私信息（PII）

**处理策略**：
| 情况 | 处理方式 |
|------|----------|
| 用户输入含敏感词 | 拒绝处理，返回提示 |
| 模型输出含敏感词 | 过滤替换或重新生成 |
| 检测到 PII | 脱敏处理 |

#### 3.4.8 对话记忆管理

**需求描述**：支持用户对话的压缩记录与长期记忆

**记忆架构**：
```
┌─────────────────────────────────────────────────┐
│                   记忆分层架构                    │
├─────────────────────────────────────────────────┤
│  工作记忆 (Working Memory)                       │
│  - 当前会话上下文                                │
│  - 最近 N 轮对话                                 │
│  - TTL: 会话结束清理                             │
├─────────────────────────────────────────────────┤
│  会话记忆 (Session Memory)                       │
│  - 单次会话完整记录                              │
│  - 会话摘要                                      │
│  - TTL: 7-30 天                                  │
├─────────────────────────────────────────────────┤
│  长期记忆 (Long-term Memory)                     │
│  - 用户偏好                                      │
│  - 重要对话摘要                                  │
│  - 向量化存储 (RAG)                              │
│  - TTL: 永久                                     │
└─────────────────────────────────────────────────┘
```

**实现方案**：
| 功能 | 技术方案 | 说明 |
|------|----------|------|
| 短期记忆 | 滑动窗口 | 保留最近 N 轮对话 |
| 对话压缩 | 摘要模型 | 长对话压缩为摘要 |
| 长期记忆 | RAG + 向量数据库 | 检索相关历史对话 |
| 用户画像 | 结构化存储 | 记录用户偏好、习惯 |

**记忆压缩流程**：
```
1. 当对话轮数超过阈值（如 20 轮）
2. 调用摘要模型生成对话摘要
3. 保留摘要 + 最近 N 轮对话
4. 将完整对话向量化存入长期记忆
5. 清理过期会话数据
```

## 八、风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| QQ 账号风控 | 机器人被封 | 优先使用官方 API，遵守使用规范 |
| 大模型 API 不稳定 | 对话功能不可用 | 支持多模型切换，配置备用模型 |
| 资讯源变更 | 推送功能失效 | 多源备份，定期维护爬虫 |
| 敏感词过滤遗漏 | 合规风险 | 多层过滤机制，定期更新词库 |
