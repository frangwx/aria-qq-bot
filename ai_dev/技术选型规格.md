# 绝区零 QQ 机器人技术方案选型与功能规格

## 技术选型决策

| 模块 | 推荐方案 | 理由 |
|------|----------|------|
| QQ 接入 | NapCat (OneBot v11) | 基于 QQ NT 客户端更稳定，社区活跃 |
| 机器人框架 | NoneBot2 | 纯 Python 异步框架，插件化架构 |
| 大模型接入 | OpenAI 兼容协议 | 国内直连，支持多模型切换 |
| 关系数据库 | SQLite | 零配置嵌入式，数据持久化 |
| 向量数据库 | ChromaDB | 纯 Python 实现，本地持久化 |
| 缓存 | 轻量级文件缓存 | 一期规模小，无需 Redis |

## 核心接口设计

### LLMProvider (大模型提供商抽象)
```python
class LLMProvider(ABC):
    @abstractmethod
    async def chat(self, messages: List[Dict], **kwargs) -> str:
        """发送对话请求，返回回复内容"""
        pass
    
    @abstractmethod
    async def get_embedding(self, text: str) -> List[float]:
        """获取文本向量嵌入"""
        pass
```

## 数据模型设计

### 用户表
```sql
CREATE TABLE users (
    user_id VARCHAR(64) PRIMARY KEY,
    qq_number VARCHAR(20) UNIQUE NOT NULL,
    nickname VARCHAR(100),
    permission_level INT DEFAULT 2,
    created_at TIMESTAMP
);
```

### 会话表
```sql
CREATE TABLE sessions (
    session_id VARCHAR(128) PRIMARY KEY,
    user_id VARCHAR(64),
    group_id VARCHAR(64),
    session_type VARCHAR(20),
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);
```

### 资讯表
```sql
CREATE TABLE news (
    id TEXT PRIMARY KEY,
    title TEXT NOT NULL,
    link TEXT NOT NULL,
    source TEXT NOT NULL,
    type TEXT NOT NULL,
    content TEXT,
    publish_time INTEGER,
    pushed INTEGER DEFAULT 0
);
```
