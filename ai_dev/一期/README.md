## 项目概述

### 项目名称
Aria QQ Bot - 绝区零 QQ 机器人助手

### 项目目标
- 构建一个稳定、可扩展的 QQ 机器人服务
- 提供绝区零游戏相关的资讯推送与查询功能
- 接入大模型实现智能对话，扮演游戏角色"爱芮"
- 支持多用户会话管理与长期记忆

### 技术栈

| 模块 | 技术方案 |
|------|----------|
| 机器人框架 | NoneBot2 |
| QQ 接入方案 | NapCat (OneBot v11) |
| 大模型接入 | OpenAI 兼容协议 |
| 向量数据库 | ChromaDB |
| 关系数据库 | SQLite |
| 缓存 | 内存缓存 |

## 关键决策记录

### 1. QQ 接入方案选择
**决策**：选择 NapCat (OneBot v11)

**理由**：
- go-cqhttp 已停止维护且风控严重
- NapCat 基于 QQ NT 客户端更稳定
- 社区活跃，文档完善

### 2. 数据库选择
**决策**：SQLite + ChromaDB

**理由**：
- SQLite 零配置，嵌入式，适合小规模部署
- ChromaDB 纯 Python 实现，无需额外服务
- 数据可随项目迁移

### 3. 资讯 API 选择
**决策**：使用米游社 painter API

**理由**：
- 官方 API，数据准确
- 支持分类（公告/活动/资讯）
- 响应格式稳定

**API 地址**：
```
https://bbs-api-static.miyoushe.com/painter/wapi/getNewsList
参数: client_type=4, gids=8, type=1/2/3
```

## 问题与解决方案

### 问题1：米游社 API 返回非标准 JSON
**现象**：JSON 解析错误

**解决**：更换 API 地址，使用 painter API 替代原 API

### 问题2：资讯链接无法访问
**现象**：生成的链接 404

**解决**：更新链接格式为 `https://www.miyoushe.com/zzz/article/{post_id}`

### 问题3：gids 配置错误
**现象**：绝区零资讯返回空数据

**解决**：绝区零 gids 应为 8，而非 52
